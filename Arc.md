INTRODUCTION AND PROJECT GOALSЦель данной технической спецификации — проектирование архитектуры для текстовой RPG-песочницы с открытым миром, глубокой социальной симуляцией и интеграцией LLM (ИИ).Данный документ представляет собой полную, развернутую архитектуру, фиксирующую протоколы общения с API нейросетей, детальную структуру хранения данных, жесткое ограничение на визуальное оформление (разработка без внешних ассетов), а также полный базовый справочник контента (Скелета).1.1. Key Architectural PrinciplesПарадигма "Скелет и Мясо" (Strict Isolation): Жесткое разделение математической логики и нарративного оформления.Code-Only Rendering (Без внешних ассетов): На этапе разработки и в базовой версии игры полностью отсутствуют спрайты, текстуры, картинки и 3D-модели. Визуализация (карта, интерфейс, иконки) создается исключительно программным кодом Godot (использование примитивов ColorRect, Panel, RichTextLabel, процедурная генерация через _draw(), а также ASCII/UTF-8 символы для иконок).Двухуровневая генерация (Macro & Micro): Контент генерируется в фоне (Macro) и JIT (Micro).Zero-Trust AI & Fallback First: Защита от мусорных ответов и падений API.Thread-Safe Determinism: Фоновая симуляция полностью изолирована от UI-потока.Zero-Allocation Policy: Оптимизация Garbage Collector (C#).Data-Driven (Отвязка контента от кода): Контент живет в JSON-файлах.2. 3. CORE MODULE & GODOT INTEGRATIONТеневая симуляция (Thread Safety): Симуляция работает с GameState_Live. Раз в секунду делается копия в GameState_Snapshot под блокировкой ReaderWriterLockSlim. UI читает данные только из Snapshot-а.Zero-Allocation: При формировании промптов конкатенация строк через + запрещена. Используются пулы и StringBuilder.Визуализация (Code-Only): Отсутствие TextureRect с загруженными изображениями (png/jpg). Вся графика строится на темах Godot (Theme), узлах контейнеров и отрисовке линий.4. ИГРОВЫЕ СИСТЕМЫ, ДАННЫЕ И МЕХАНИКА (THE SKELETON)4.1. Боевая система (Stat-Driven Combat Framework)Бой полностью изолирован от нейросети. Акцент смещен на детерминированные характеристики персонажа. Случайность дает лишь вариативность (±10% урона) и шанс критического удара/промаха.Абстрактное позиционирование: Зоны Авангард (Vanguard) и Арьергард (Rearguard). Враг ближнего боя не может атаковать Арьергард, пока не уничтожен Авангард.Очередь и тайм-менеджмент: Инициатива рассчитывается из стата Ловкости (DEX). На ход дается жесткий таймер (10 сек). Опоздание — авто-защита.Отступление (Fleeing): Математическая проверка: DEX игрока + Кардио против DEX самого быстрого врага. Провал означает пропуск хода и удар в спину.4.2. Состояние проигрыша (Death & Failure State)Падение здоровья (HP) до 0 приводит к нарративному сдвигу и экономическому штрафу.Механика Respawn: Simulation проматывает игровое время на 1-3 дня вперед. Персонаж "просыпается" в ближайшей безопасной зоне (Храм/Таверна).Штрафы: Потеря % золота, резкое снижение прочности надетой экипировки, получение "Травмы" (дебафф статов).Интеграция ИИ: При пробуждении LLMClient генерирует короткий текст о том, как вас спасли.4.3. Износ экипировки (Durability)В DataManager внедрены параметры CurrentDurability и MaxDurability.Износ: Снижение прочности оружия при атаках (особенно по бронированным целям), брони — при получении урона.Поломка: При достижении 0 предмет не исчезает, но его параметры обнуляются (до 1), пока он не будет восстановлен у кузнеца.4.4. Глобальная симуляция: Королевства, Города и МаршрутыМир — это взвешенный граф. Иерархия: Королевство -> Город -> Деревня -> Точка интереса (POI). Во время путешествия по маршруту-дороге движок бросает кубики на Случайные Встречи (Random Encounters) на основе параметра Опасности дороги и Удачи игрока.4.5. Система квестов и их выдачи (Quest Pipeline)Сборка Скелета: QuestManager собирает математический граф. Пример: [Node A: Идти в Пещеру] -> [Node B: Убить Бандита] -> [Node C: Получить Золото].Генерация Мяса: Скелет отправляется в ИИ. ИИ возвращает JSON с названием, лорной причиной и текстом для NPC.Исполнение: После принятия квеста ИИ отключается. QuestManager локально следит за счетчиком.4.6. Крафт и GOAP (Поведение NPC)CraftingEngine: Валидирует наличие ингредиентов. Рассчитывает шанс успеха на основе Интеллекта. При успехе запрашивает у ИИ название и описание предмета.GOAPEngine: Математически рассчитывает рутину NPC (Голод = 80 -> Цель: Поесть). Результат передается в ИИ как контекст для логичных ответов в диалоге.5. AI MANAGEMENT & API INTERACTION (ВЗАИМОДЕЙСТВИЕ С ИИ)Протокол общения с ИИ жестко стандартизирован. Игра ожидает только валидный JSON.5.1. Структура API запроса и ответаЗапрос (User Payload):{
  "context": {
    "location": "Таверна",
    "npc_state": {"name": "Гаррик", "profession": "Трактирщик", "goap_goal": "Обслуживать клиентов"},
    "player_action": "[Убеждение] Налей мне эля бесплатно."
  }
}
Ожидаемый ответ (JSON Schema):{
  "thoughts": "Игрок манипулирует. Репутация высока. Я соглашусь.",
  "spoken_text": "Держи кружку. Но это в последний раз!",
  "action_triggers": [
    {"type": "give_item", "id": "drink_ale", "amount": 1}
  ]
}
5.2. Защита от Prompt Injection (InputSanitizer.cs)Санитаризация: Ввод прогоняется через blacklists и RegEx (вырезаются мета-команды ignore instructions).System Prompt Armor: Системные инструкции (P0) требуют игнорировать команды из блока Player_Input.Валидация Скелета (Абсолютная защита): Если ИИ в action_triggers укажет выдать 10000 золота, GraphValidator отклонит этот ответ, так как он нарушает локальные лимиты узла.5.3. Приоритеты Prompt и Долгосрочная память (RAG)Распределение хард-лимита токенов:System Rules (P0): ~300 токенов. Жесткие рамки.Current Objective & GOAP (P1): ~200 токенов. Математическая цель NPC или квеста.World State (P2): ~500 токенов. Время суток, травмы, погода.RAG Context / Память (P3): ~600 токенов. VectorDB делает поиск старых семантически похожих фактов (игрок украл яблоко 10 дней назад).Recent Chat (P4 - FIFO): Оставшиеся токены на прямой лог беседы.6. Пользовательские директории (Godot user://)user://saves/ — БД сохранений.user://cache/ — Кэш ИИ.user://exported_worlds/ — Префабы (Скелеты макро-мира) в формате JSON.user://config/ — Настройки UI, ключи (зашифрованные).6.2. База данных сохранений (SQLite)Каждое сохранение — это файл .db. Позволяет подгружать мир чанками.PlayerState: Базовые статы, уровень, координаты, время.Inventory: Таблица связи [player_id] -> [item_id] -> [quantity] -> [current_durability].WorldNodes: Состояние открытых локаций.EntitiesState: Состояние NPC. Если хеш (ID NPC + стейт) найден в БД, текст берется локально (0 мс задержка, 0 потраченных токенов).7. UX ARCHITECTURE & ДЕТАЛИЗАЦИЯ ИНТЕРФЕЙСА7.1. Масштабирование и Защита версткиGodot Anchors & Containers: Запрет на абсолютное пиксельное позиционирование.Anti-Overlap: Весь ИИ-текст обернут в ScrollContainer с autowrap_mode. Текст никогда не сдвинет соседние окна.Resizing: Окна DraggableWindow автоматически пересчитываются пропорционально при смене разрешения экрана.7.2. Асинхронные конфликты (Task Cancellation)При отправке запроса в ИИ передается CancellationToken. Если игрок закроет окно DialogBox не дождавшись ответа, токен отменит запрос, предотвратив попытку ИИ записать данные в выгруженный UI (и избежав краша).7.3. Навигация: КартыLocalMap: Узловая структура комнаты/подземелья.GlobalMap: Поддержка Zoom (колесиком). Оснащена выпадающим списком-поиском. Выбор открытой деревни в списке мгновенно центрирует камеру.7.4. Меню Персонажа (CharSheetUI)Детально отражает состояние "Скелета". Блоки: Аватарка/Уровень, Основные статы (STR/DEX), Вторички (Уворот/Мент.Сопр), Активные травмы и баффы с таймерами.7.5. Инвентарь и Tooltip СравнениеВкладки: Оружие, Броня, Материалы, Еда/Зелья, Квестовые предметы. Поддержка фильтров и сортировки.Hover Compare: При наведении мыши на броню в инвентаре появляются два Tooltip окна: надетая вещь и наведенная. Разница подсвечивается зеленым/красным цветом.7.6. Меню КрафтаДерево рецептов слева, требуемые материалы по центру (красный текст при нехватке), и прогноз шанса успеха справа.8. PRODUCTION & ERROR HANDLINGErrorManager: Circuit Breaker. Если API дает 429/500 ошибки — включается Exponential Backoff. Максимум 3 попытки, затем Fallback-заглушка из БД.CostTracker: Локальный токенизатор считает расходы игрока.TelemetryLogger: Сброс "галлюцинаций" (непрошедших валидацию ИИ-ответов) в локальный текстовый лог для дальнейшего анализа.9.
